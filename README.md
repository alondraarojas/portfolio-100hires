# portfolio-100hires
Portfolio project for 100Hires application
Overview

This repository documents the process I followed to complete the portfolio exercise requested by 100Hires as part of their selection process. Rather than a polished, linear success story, this README reflects the actual journey, trial and error, and adaptation, which I believe says more about how I approach unfamiliar problems than a perfect outcome would.

Phase 1: Discovery

I started by reading the email carefully and quickly realized I didn't know how to do any of what was being asked, since I wasn't familiar with any of the platforms mentioned (Cursor, Claude Code, Codex). Rather than being discouraged, I treated this as the actual starting point of the exercise and began researching each tool from scratch.

Phase 2: Building What Was Requested

(1) Installed Cursor IDE
(2) Created a Cursor account
(3) Reopened the app to continue setup
(4) Tried to install both Claude Code and Codex through the Extensions marketplace, exactly as the instructions described — but neither one appeared in the search results, no matter how I searched

Phase 3: Adaptation

Once the extensions didn't show up, I leaned on Claude AI for guidance and also looked through several YouTube tutorials. None of them matched what I was seeing on my screen, it became clear that Cursor had gone through a fairly recent update, and the Extensions flow described in the instructions no longer existed in the current version.

At first I assumed I had installed something incorrectly. But after digging through YouTube comment sections, I found other users reporting the exact same issue — including on a video posted only 11 days earlier that already showed a different interface than what I had. That confirmed the problem wasn't on my end; the tooling had simply moved faster than the instructions.

PHase 4: Solutions

I worked through this with Claude AI step by step, including an attempt to install Claude Code directly through the macOS terminal (via Spotlight), running the install command manually. That attempt didn't work either.

Eventually, I found this YouTube video that showed how to retrieve the installation code directly from Claude's own platform. Following that updated procedure, I was able to install Claude Code successfully using that code.

For Codex, I downloaded it directly from its official source without further complications.

Conclusions

I'm honestly not sure if every solution I landed on was the "correct" or most elegant one, but it's what I was able to figure out with the resources available to me. Along the way, I also searched TikTok for quicker, more visual walkthroughs, and I'm still actively looking for a way to get both tools properly recognized inside Cursor itself. I managed to install Claude Code and Codex independently, but surfacing them within Cursor's own interface is something I haven't fully solved yet; and I'm continuing to work on it.

I'd genuinely welcome feedback on this, whatever the outcome of the application turns out to be. This exercise opened my eyes to how much there still is for me to learn about development tooling, and how valuable it is to stay adaptable when documentation doesn't match reality.
